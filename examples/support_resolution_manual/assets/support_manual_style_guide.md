# Support Manual Style Guide

- Use concise operational language.
- Separate internal agent notes from customer-facing templates.
- Keep customer-facing language empathetic and precise.
- Do not promise outcomes that product docs do not support.
- Include escalation triggers before advanced troubleshooting.
- Preserve product setting names and UI labels only when they appear in product
  docs or support records.
- Put policy uncertainty or outdated macros in `Maintenance Notes`.
