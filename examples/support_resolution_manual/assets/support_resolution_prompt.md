# Task

Create an official support resolution manual from user inquiry tickets, support
response notes, product references, and approved macros.

Use the support records as evidence for what customers ask and what support
agents actually do. Use product documentation to avoid contradicting official
behavior. Use approved macros only when they remain accurate.

Write the manual for support agents handling live customer conversations. The
manual must be specific enough to follow, but must not expose internal-only
notes to customers except where a customer-facing response template is
explicitly requested.

Do not invent product capabilities, refund rules, SLA promises, or escalation
owners that are not present in the sources.
