# Incident Runbook Style Guide

- Use imperative language for steps.
- Keep active-incident steps short enough to execute under pressure.
- Separate detection, mitigation, recovery, and follow-up.
- Include decision points and escalation criteria.
- Avoid blame-oriented wording.
- Preserve exact metric names, dashboard names, queue names, and service names
  only when they appear in the source records.
- Put unknowns and missing ownership in `Maintenance Notes`.
