# Task

Create an official incident response runbook from the supplied incident event
logs, response-room notes, postmortems, and existing runbooks.

Use the source records as evidence. Prefer repeated operational patterns over
one-off guesses. If evidence conflicts, describe the conflict and choose the
safer operating procedure.

The runbook must be actionable during an active incident. Write it for on-call
engineers and incident commanders.

Do not invent system behavior, owners, escalation targets, thresholds, or
tooling that are not present in the sources. Mark unresolved gaps in
`Maintenance Notes`.
