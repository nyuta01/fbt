---
title: OpenTelemetry export
description: Export local fbt run results as OTLP/JSON traces.
---

`fbt export otel` emits local-first OTLP/JSON traces from fbt run results.

```sh
fbt export otel \
  --project-dir /tmp/fbt-lineage-demo \
  --output /tmp/fbt-lineage-demo/target/telemetry/otel.json
```

The command prints a short summary:

```text
Export: otel
  Format  OpenTelemetry OTLP/JSON traces
  Output  /tmp/fbt-lineage-demo/target/telemetry/otel.json
  Spans   4
  Next    post this JSON to an OTLP-compatible collector or store it as evidence
```

The export includes invocation spans, transform spans, usage attributes, policy
decision references, and safe runner events.

Raw source content and known marker secrets are redacted by conformance tests.

## Typical backends

- Jaeger
- Tempo and Grafana
- any collector that accepts OTLP/JSON after conversion or forwarding

For a running OTLP HTTP endpoint, the opt-in smoke target posts generated fbt
traces:

```sh
FBT_OTLP_TRACES_URL=http://localhost:4318/v1/traces make standard-backend-smoke
```
