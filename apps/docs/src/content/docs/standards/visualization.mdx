---
title: Visualization
description: Use standard lineage and trace tools instead of a custom graph UI.
---

fbt deliberately avoids owning a custom lineage UI in core.

Use standard outputs:

| Need | fbt output | Common target |
|---|---|---|
| Dataset/job lineage | OpenLineage NDJSON | Marquez |
| Catalog integration | OpenLineage events | OpenMetadata ingestion |
| Execution traces | OTLP/JSON | Jaeger, Tempo, Grafana |
| Local audit | artifact inspection | `fbt artifact`, `fbt diff` |

This keeps fbt local-first while still letting teams use existing observability
and catalog systems.

Create a local fixture and export both files:

```sh
fbt init /tmp/fbt-viz-knowledge --template support
fbt build --project-dir /tmp/fbt-viz-knowledge --select case_summaries
fbt build --project-dir /tmp/fbt-viz-knowledge --select weekly_support_insights
mkdir -p /tmp/fbt-viz-knowledge/target/lineage /tmp/fbt-viz-knowledge/target/telemetry
fbt export openlineage \
  --project-dir /tmp/fbt-viz-knowledge \
  --output /tmp/fbt-viz-knowledge/target/lineage/openlineage.ndjson
fbt export otel \
  --project-dir /tmp/fbt-viz-knowledge \
  --output /tmp/fbt-viz-knowledge/target/telemetry/otel.json
```

The automated opt-in fixture is:

```sh
make standard-backend-smoke
```

With no backend variables, it verifies the OpenLineage and OTLP/JSON files
locally and skips network ingestion.

Post OpenLineage events to a local Marquez API:

```sh
MARQUEZ_URL=http://localhost:5000
while IFS= read -r event; do
  curl -sS -X POST "$MARQUEZ_URL/api/v1/lineage" \
    -H 'Content-Type: application/json' \
    --data "$event" >/dev/null
done < /tmp/fbt-viz-knowledge/target/lineage/openlineage.ndjson
```

Post OTLP/JSON traces to an OTLP HTTP endpoint:

```sh
curl -sS -X POST http://localhost:4318/v1/traces \
  -H 'Content-Type: application/json' \
  --data-binary @/tmp/fbt-viz-knowledge/target/telemetry/otel.json
```

Or run both backend posts through the opt-in smoke target:

```sh
FBT_MARQUEZ_URL=http://localhost:5000 \
FBT_OTLP_TRACES_URL=http://localhost:4318/v1/traces \
FBT_STANDARD_EVIDENCE_DIR=/tmp/fbt-standard-evidence \
make standard-backend-smoke
```

Then use Marquez, Jaeger, Tempo, Grafana, or OpenMetadata as the visualization
surface. Any screenshot in fbt docs should come from one of those actual
backends after running the recipe, not from a custom fbt diagram.
