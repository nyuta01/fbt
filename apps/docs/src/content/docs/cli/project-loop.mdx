---
title: Project loop
description: Parse, plan, build, inspect, and document a project.
---

The normal fbt loop is:

```sh
fbt doctor
fbt plan
fbt build
fbt artifact explain <artifact>
fbt docs generate
```

## Check

`fbt doctor` reads `fs_project.yml` plus source, transform, asset, policy,
eval, and runner YAML files. It checks state and runner readiness before any
build work starts.

## Plan

`fbt plan` compares the manifest and local state. It reports dirty artifacts,
clean artifacts, missing upstream artifacts, and skipped work.

## Build

`fbt build` invokes external runners, validates output candidates, records run
results, commits artifact versions, and runs configured evals.

## Inspect

`fbt artifact path`, `fbt artifact show`, `fbt artifact history`, and
`fbt artifact explain` show where artifacts live, which version is current,
and why a transform will run or skip.

## Docs

`fbt docs generate` optionally writes project-local Markdown docs with artifact lineage,
eval results, and policy decision references.
