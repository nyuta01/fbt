---
title: Project loop
description: Check, plan, build, inspect, and export a project.
---

The normal fbt loop is:

```sh
fbt doctor
fbt plan
fbt build
fbt artifact explain <artifact>
fbt export openlineage
```

## Check

`fbt doctor` reads `fs_project.yml` plus source, transform, asset, policy,
eval, and runner YAML files. It checks state and runner readiness before any
build work starts.

## Plan

`fbt plan` compares the manifest and local state. It reports dirty artifacts,
clean artifacts, missing upstream artifacts, and skipped work.

Use `fbt plan --force` to preview a deliberate rebuild of selected clean
transforms. It stays read-only and reports `reason: forced rebuild`.

## Build

`fbt build` invokes external runners, validates output candidates, records run
results, commits artifact versions, runs configured evals, and writes the build
receipt. The command name follows build-tool semantics: declared file inputs
produce declared file outputs.

Use `fbt build --force` to regenerate selected clean transforms. Force does not
bypass upstream, confidence, policy, or output-boundary checks.

## Inspect

`fbt artifact path`, `fbt artifact show`, `fbt artifact history`, and
`fbt artifact explain` show where artifacts live, which version is current,
and why a transform will run or skip.

## Export

`fbt export openlineage` and `fbt export otel` write standard records that can
be loaded into lineage, catalog, and trace tools.
