---
title: Inspection
description: Commands for explaining artifacts and runs.
---

fbt is designed for inspectability. These commands are useful after a build:

```sh
fbt artifact path case_summaries
fbt artifact show case_summaries
fbt artifact history case_summaries
fbt artifact explain weekly_support_insights
fbt artifact retention
fbt diff case_summaries --against previous
```

## Explain a plan decision

```sh
fbt artifact explain weekly_support_insights --project-dir knowledge_ops
```

This reports why an artifact is dirty, clean, blocked, or skipped.
It includes the producing transform, current version, previous run, source or
upstream artifact inputs, changed fingerprints when available, dirty or blocked
reasons, and the next command to run.

## Inspect local retention

```sh
fbt artifact retention --project-dir knowledge_ops
```

This read-only report shows local state size, immutable artifact size, run
records, current versions, historical versions, and missing storage references.
fbt keeps all versions by default; archive `.fbt/state` and `.fbt/artifacts`
together when an external retention process is needed.

For high-volume guidance, run `make retention-high-volume-smoke` from a source
checkout or read `docs/examples/high-volume-retention.md`. The fixture creates
multiple artifact versions and verifies that retention inspection removes no
files.

## Inspect generated files

```sh
fbt artifact path case_summaries --project-dir knowledge_ops
```

Use the path command when a user wants to open or hand off the generated file.

## Inspect lineage

Use standard export commands when lineage needs to leave the local project:

```sh
fbt export openlineage --project-dir knowledge_ops
fbt export otel --project-dir knowledge_ops
```
