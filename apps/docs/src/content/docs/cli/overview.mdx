The `fbt` CLI is the main interface for local projects.

## Core commands

| Command | Purpose |
|---|---|
| `fbt init` | Create a project from a template |
| `fbt doctor` | Check project readiness and runner setup |
| `fbt plan` | Show what will run, skip, or block |
| `fbt build` | Execute selected transforms |
| `fbt diff` | Compare artifact versions |
| `fbt artifact` | Inspect paths, versions, and explanations |
| `fbt export` | Export standard lineage and trace data |

Parsing, state writes, evals, and runner diagnostics happen inside these
commands. The public surface stays small so daily operation is a short loop
instead of a toolbox of debug subcommands.

## Global flags

```sh
fbt plan --project-dir knowledge_ops --select tag:support
fbt build --project-dir knowledge_ops --select case_summaries --json
```

Common flags:

- `--project-dir PATH`
- `--state-dir PATH`
- `--select EXPR`
- `--json`
