---
title: Authoring contract
description: Contract external runner authors must satisfy.
---

An fbt runner is an external command that speaks the stdio JSON-RPC protocol.
The command may wrap a provider SDK, CLI agent, document converter, shell
script, or internal service.

The smallest copyable starting point is
`examples/runner_adapter_scaffold/bin/fbt-runner-example`. It uses only the
Python standard library and passes strict conformance.

## Runner must

- implement `initialize`
- advertise protocol version and capabilities
- implement `fbt/runTransform`
- write output files only under the assigned `work.outputs` directory
- return output candidates with artifact names and paths
- emit safe, redacted progress and usage events
- fail closed when policy cannot be translated safely

## CLI-agent adapters

Adapters for tools such as Claude Code or Codex should run the agent in a
staging workspace, translate fbt policies into agent constraints, collect final
files, and copy only the intended output candidates into `work.outputs`.

## Conformance

Use the conformance harness before publishing a runner:

```sh
FBT_RUNNER_CONFORMANCE_COMMAND="your-runner command" make runner-conformance
```

Check the scaffold itself with:

```sh
python3 tests/runner-conformance/run.py \
  --runner-command examples/runner_adapter_scaffold/bin/fbt-runner-example \
  --strict \
  --agent-adapter
```

For installed adapters, use `make runner-adapter-smoke` with
`FBT_RUNNER_ADAPTER_SMOKE_MATRIX` to run conformance plus generated-project
`doctor` and `plan` checks without adding provider SDKs to fbt core.
