## Install with the script

The easiest path on macOS, Linux, or Windows Git Bash is:

```sh
curl -fsSL https://raw.githubusercontent.com/nyuta01/fbt/main/install.sh | sh
fbt version
```

The installer detects your OS and architecture, downloads the matching GitHub
Release archive, verifies `SHA256SUMS`, and installs `fbt` to
`$HOME/.local/bin` by default.

Pin a version or directory when needed:

```sh
curl -fsSL https://raw.githubusercontent.com/nyuta01/fbt/main/install.sh | \
  sh -s -- --version v0.2.1 --dir "$HOME/.local/bin"
```

## Download a release binary manually

The current core CLI release is published on GitHub:

```text
https://github.com/nyuta01/fbt/releases/tag/v0.2.1
```

Download the archive for your platform and verify it with `SHA256SUMS`.

```sh
shasum -a 256 -c SHA256SUMS
```

Then put the `fbt` binary on `PATH` and verify it:

```sh
fbt version
fbt init /tmp/fbt-install-check --template support
fbt doctor --project-dir /tmp/fbt-install-check
```

On Windows without Git Bash, download the `windows_amd64.zip` or
`windows_arm64.zip` archive from the release page, compare its `Get-FileHash`
SHA-256 value with `SHA256SUMS`, expand the archive, and add the directory
containing `fbt.exe` to `PATH`.

## Build from source

```sh
git clone https://github.com/nyuta01/fbt.git
cd fbt
make build
./bin/fbt version
```

## Install official adapters

fbt core does not include provider SDKs or agent runtimes. Install only the
runner commands you need:

```sh
go install github.com/nyuta01/fbt/adapters/command/cmd/fbt-runner-command@main
go install github.com/nyuta01/fbt/adapters/openai/cmd/fbt-runner-openai@main
go install github.com/nyuta01/fbt/adapters/codex-cli/cmd/fbt-runner-codex-cli@main
go install github.com/nyuta01/fbt/adapters/claude-code/cmd/fbt-runner-claude-code@main
```

For reproducible environments, pin a commit or an adapter module tag when one
is released. The adapter command and fbt core communicate through the runner
protocol; adapter installation stays out-of-band.

## Verify adapters

From a source checkout, verify official adapters without live provider calls:

```sh
make official-adapter-smoke
make adapter-install-smoke
```

For installed commands, use the opt-in smoke matrix. This validates protocol
conformance, `doctor`, and `plan` without a live build:

```sh
export OPENAI_API_KEY=...
FBT_RUNNER_ADAPTER_SMOKE_MATRIX='openai.responses|llm|markdown|fbt-runner-openai responses|OPENAI_API_KEY|false' \
make runner-adapter-smoke
```

Add `FBT_RUNNER_ADAPTER_SMOKE_BUILD=1` only when you intentionally want to call
the real provider or agent and commit a temporary artifact.

## Verify the checkout

```sh
make verify
```

The verification gate runs harness checks, docs checks, Go formatting and
tests, CLI smoke tests, practical example smoke tests, runner conformance,
product conformance, and a local release binary smoke.
