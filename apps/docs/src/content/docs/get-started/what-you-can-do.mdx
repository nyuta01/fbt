title: What you can do today
description: Concrete workflows implemented by fbt v0.1.0.
---

Use fbt when the important work already lives in files and you need a
repeatable way to turn those files into versioned artifacts.

fbt v0.1.0 does not try to be the model, OCR tool, document converter, or
agent runtime. It gives generated files a build receipt: the selected sources,
runner, output version, checks, local state, lineage, and standard exports.

The proof in these docs is the command transcript and the generated files. fbt
does not publish a custom quickstart graph UI; when you need graph
visualization, export OpenLineage or OTel records into standard tools.

## Implemented Workflows

| You want to | Use | Result |
|---|---|---|
| Prove the local control-plane loop | `fbt init knowledge_ops --template support` | A runnable support project with demo LLM and agent runners. |
| See what will happen before running it | `fbt doctor`, `fbt plan` | Readiness checks plus run or blocked reasons. |
| Build a first artifact from source files | `fbt build --select case_summaries` | Markdown output in `target/artifacts`, an immutable copy in `.fbt/artifacts`, and a local receipt in `.fbt/state`. |
| Wrap existing document tools | `examples/markdown_toolchain` | remark/Pandoc-style command wrappers produce artifacts while fbt records receipts and lineage. |
| Compose with data tools | `examples/data_tool_interop` | dbt and DataChain output files become fbt sources for a versioned human brief. |
| Explain where a file came from | `fbt artifact history`, `fbt artifact explain` | Source refs, transform, runner, model, digest, eval, confidence, and policy state. |
| Send lineage to existing tools | `fbt export openlineage` | OpenLineage RunEvent NDJSON for Marquez and OpenMetadata ingestion paths. |
| Send execution telemetry to trace tools | `fbt export otel` | OTLP/JSON spans for OpenTelemetry-compatible trace backends. |
| Use a real model or CLI agent | Replace the configured runner command | The same build graph calls OpenAI, Claude Code, Codex, Gemini, or another compatible adapter. |

## What It Does Not Do

| fbt does not | What to use instead |
|---|---|
| Call every model provider from core | External runners that speak the fbt stdio JSON-RPC protocol. |
| Run semantic model judges in core | External judge transforms that produce report artifacts. |
| Convert PDFs, Word files, or images by itself | A document conversion or OCR runner. |
| Host a lineage graph UI | OpenLineage, OpenMetadata, Jaeger, Tempo, Grafana, or generated local docs. |
| Replace your knowledge base or ticket system | Export or copy the generated artifact into that system through your existing publishing process. |
| Own human approvals | Use Git, PRs, CI, release tooling, or your knowledge-base workflow. |
| Make stochastic model output fully reproducible | Keep immutable artifact versions, descriptors, evals, and lineage for audit. |

## The Fastest Real Check

Run the quickstart when you want to verify fbt's control-plane loop on your
machine. It is a fixture, not a realistic support-manual workflow.

The quickstart answers this question:

> Can fbt check a local project, run configured runners, version artifacts,
> connect upstream artifacts to downstream transforms, and export local
> lineage/telemetry?

Run it and inspect these files:

```sh
fbt init knowledge_ops --template support
fbt build --project-dir knowledge_ops --select case_summaries
fbt build --project-dir knowledge_ops --select weekly_support_insights
```

The run writes:

```text
knowledge_ops/target/artifacts/support/case_summaries/index.md
knowledge_ops/target/artifacts/support/weekly_insights.md
knowledge_ops/.fbt/state/run_results.jsonl
knowledge_ops/.fbt/state/artifact_versions.json
```

Then export standard records:

```sh
mkdir -p knowledge_ops/target/lineage knowledge_ops/target/telemetry
fbt export openlineage --project-dir knowledge_ops --output knowledge_ops/target/lineage/openlineage.ndjson
fbt export otel --project-dir knowledge_ops --output knowledge_ops/target/telemetry/otel.json
```

The quickstart page shows the exact command output from this flow.
